O objetivo estimar a distribuio a posteriori da correlao de duas variveis com distribuio normal padro usando o algoritmo de MetropolisHastings.

Para isso vamos definir um valor de correlao $\rho$ e gerar uma amostra com $N$ elementos da distribuio normal bivarivel:

$(X, Y) \sim \mathcal{N}(\mu; \Sigma)$

onde $\mu = (\mu_X,\mu_Y)$ e $\Sigma = \begin{equation*} \mathbf{}\left[\begin{matrix} \sigma^2_X & \rho \sigma_X \sigma_Y\\ \rho \sigma_X \sigma_Y & \sigma^2_Y \end{matrix}\right] \end{equation*} $.

No nosso caso: $(X, Y) \sim \mathcal{N}\left((0,0); \begin{equation*} \mathbf{}\left[\begin{matrix} 1 & \rho\\ \rho & 1 \end{matrix}\right] \end{equation*}\right)$.

Com esses valores a funo de verossimilhana fica sendo:

$$p((X,Y) ~|~ \rho) = \prod_{i=1}^n \frac{1}{2\pi\sqrt{1-\rho^2}}~\exp\left\{-\frac{1}{2(1-\rho^2)}\left[x_i^2-2\rho x_i y_i + y_i^2\right]\right\}$$

Para a priori vamos usar a priori de Jeffreys do parmetro da normal bivarivel (Gelman et al., 1995): $p(\rho)=\frac{1}{|\Sigma|^{3/2}}$.

No nosso caso: $p(\rho)=\frac{1}{(1-\rho^2)^{3/2}}$. Observe que a integral da priori no converge, logo uma priori degenerada.

Com isso nossa distribuio a posteriori fica:

$$p(\rho ~|~ (X,Y)) \propto \frac{1}{(1-\rho^2)^{3/2}}~\prod_{i=1}^n \frac{1}{2\pi\sqrt{1-\rho^2}}~\exp\left\{-\frac{1}{2(1-\rho^2)}\left[x_i^2-2\rho x_i y_i + y_i^2\right]\right\}$$

Vamos implementar a funo que calcula esses valores para ver o que acontece:

def prod(rho, X, Y):
    P = 1
    for w in zip(X, Y):
        P *= 1 / (2*np.pi * np.sqrt(1-rho*rho))
        P *= np.exp( (w[0]*w[0] - 2*rho*w[0]*w[1] + w[1]*w[1]) / (-2*(1-rho*rho)) )
    return P

def prob(rho, X, Y):
    return pow(1-rho*rho, -3/2)*prod(rho, X, Y)

def ln_prod(rho, X, Y):
    n = len(X)
    P = -n * (np.log(2*np.pi) + np.log(1-rho*rho) / 2)
    
    S = 0
    for w in zip(X, Y):
        S += w[0]*w[0] - 2*rho*w[0]*w[1] + w[1]*w[1]
    S /= 2 * (1-rho*rho)
    
    return P - S

def ln_prob(rho, X, Y):
    return -3*np.log(1-rho*rho)/2 + ln_prod(rho, X, Y)

LL = [ln_prod(i, X, Y) for i in np.arange(-0.99, 0.99, 0.01)]

O algoritmo¶

Vamos usar o MetropolisHastings para gerar a nossa amostra de $\rho$. O algoritmo vai provocando perturbaes aleatrias no valor de $\rho$ e decide se esse novo valor vai ser aceito ou rejeitado. Usaremos uma uniforme para fazer a perturbao no valor de $\rho$:

$$\rho \sim \mathcal{U}(\rho^{i-1}-\varepsilon; \rho^{i-1}+\varepsilon)$$

Devemos aceitar ou rejeitar o valor de $\rho$ de uma forma que d mais peso para maiores valores de verossimilhana. Para isso calcularemos a razo da verossimilhana anterior com a nova verossimilhana:

$$\alpha = \frac{p(\rho^{(i)} ~|~ (X,Y))}{p(\rho^{(i-1)} ~|~ (X,Y))}$$

Se $\alpha$ for maior que 1 eu aceito o candidato a $\rho$, se for entre $0$ e $1$ eu vou aceitar o candidato com probabilidade $\alpha$.

In [8]:
eps = 0.03

rho = 0
all_vals = []

burn = 400
use = 2000

for i in range(burn + use):
    
    # Gera o candidato
    candidate = np.random.uniform(rho-eps, rho+eps)
    
    # Regulariza o valor
    if (candidate < -1):
        candidate = -1+eps
    if (candidate > 1):
        candidate = 1-eps

    # Calcula o alpha
    ln_alpha = ln_prob(candidate, X, Y) - ln_prob(rho, X, Y)
    alpha = np.exp(ln_alpha)
    
    # Calcula o alpha sem usar o ln (no funciona)
    #alpha = prob(candidate, X, Y) / prob(rho, X, Y)

    # Descarta ou aceita o candidato
    u = np.random.uniform(0, 1)
    if (u < alpha):
        rho = candidate
    
    all_vals += [rho]

amostra = all_vals[burn:]
    
plt.plot(all_vals);

Convergiu bem rpido para o valor definido de $0,8$, e descartando os primeiros valores podemos ter uma aproximao da densidade de $\rho$.
